# Heritage Lens Agent — Lessons Learned

## Session: 2026-06-01

### 1. File Permissions Matter
- **Mistake**: Tried to edit `requirements.txt` without checking permissions first. Got `EACCES: permission denied`.
- **Root cause**: File was owned by `root` instead of `heritage` user.
- **Fix**: `sudo chown heritage:heritage <file>` before editing.
- **Lesson**: Always `ls -la` before editing files. Don't assume ownership matches the working directory.

### 2. Git State Can Be Messy After Fixes
- **Mistake**: After restoring `chroma_db/` from zip, git showed nested paths (`chroma_db/chroma_db/`) and uncommitted changes.
- **Root cause**: Unzipped to wrong directory level, creating nested structure.
- **Fix**: Careful `mv` and cleanup of intermediate directories.
- **Lesson**: When extracting archives, verify the target structure before committing.

### 3. Process Management Requires Care
- **Mistake**: Killed Streamlit process but restart didn't work cleanly via `exec` background.
- **Root cause**: Using `&` in `exec` doesn't properly daemonize; systemd was the right way.
- **Fix**: Let systemd handle the service (`systemctl restart heritage-streamlit`).
- **Lesson**: When a systemd service exists, use it. Don't fight the init system.

### 4. Check Existing Implementation Before Adding Features
- **Mistake**: Asked about "upload button" when it was already implemented in the sidebar.
- **Root cause**: Didn't read the full `app.py` before responding.
- **Lesson**: Always read the current file state before assuming something is missing.

### 5. Color/Design Specs Live in Wireframes, Not Just Code
- **Mistake**: Panel 3 was dark blue when wireframe specified light blue.
- **Root cause**: Didn't reference the wireframe images (`heritage_lens_wireframe.jpg`, `Gemini_Sketchframe.png`) before making changes.
- **Fix**: Used `image` tool to analyze wireframes, then corrected colors.
- **Lesson**: Design assets in the workspace are authoritative. Check them before changing UI.

### 6. Requirements.txt Version Drift
- **Mistake**: `redis` was listed twice with different versions (`>=4.0.0` and `>=4.5.0`).
- **Root cause**: Added redis to core section without noticing it already existed in a dedicated section.
- **Fix**: Deduplicated and bumped to `>=4.5.0`.
- **Lesson**: Read the entire file before adding dependencies. Search for existing entries first.

### 7. Docker vs Host Services
- **Observation**: Streamlit runs as systemd service, not in Docker (intentionally, per `docker-compose.yml` comment).
- **Lesson**: Infrastructure comments in config files are meaningful. Read them to understand architecture decisions.

---

## General Principles

1. **Inspect before acting** — `ls -la`, `git status`, `read` the file.
2. **Use the right tool for the job** — systemd for services, not manual process juggling.
3. **Design assets are source of truth** — wireframes, PRDs, and mockups override assumptions.
4. **Keep requirements clean** — deduplicate, don't append blindly.
5. **Verify after fix** — run the health check again, don't assume.
